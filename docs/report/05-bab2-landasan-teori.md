## 2.1 Teori Umum

### 2.1.1 Pengertian Website

Website merupakan kumpulan halaman-halaman yang saling terhubung dan dapat diakses melalui jaringan internet menggunakan protokol HTTP (*Hypertext Transfer Protocol*) atau HTTPS (*Hypertext Transfer Protocol Secure*). Menurut Sucipto (2018), website adalah suatu dokumen yang ditulis dalam format HTML (*Hypertext Markup Language*) dan dapat diakses melalui peramban web (*web browser*) seperti Google Chrome, Mozilla Firefox, maupun Microsoft Edge.

Website memiliki beberapa karakteristik utama, yaitu:

1. **Aksesibilitas** — Website dapat diakses dari mana saja selama pengguna terhubung ke jaringan internet, tanpa dibatasi oleh lokasi geografis maupun waktu.
2. **Interaktivitas** — Website modern memungkinkan interaksi dua arah antara pengguna dan sistem melalui formulir, tombol, dan elemen antarmuka lainnya.
3. **Dinamis** — Website dinamis mampu menampilkan konten yang berubah-ubah sesuai dengan data yang tersimpan dalam basis data atau berdasarkan input pengguna.
4. **Skalabilitas** — Website dapat dikembangkan dan ditingkatkan kapasitasnya seiring dengan bertambahnya jumlah pengguna dan kebutuhan fungsionalitas.

Berdasarkan fungsinya, website dapat diklasifikasikan menjadi tiga jenis, yaitu website statis yang menampilkan konten tetap, website dinamis yang kontennya diperbarui secara otomatis, dan website interaktif yang memungkinkan partisipasi aktif pengguna (Sutarman, 2019).

> **Referensi:** Sucipto, A. (2018). *Teknologi Informasi dan Komunikasi*. Yogyakarta: Gosyen Publishing. | Sutarman. (2019). *Membangun Aplikasi Web dengan PHP dan MySQL*. Jakarta: Bumi Aksara.

### 2.1.2 Pengertian Sistem Informasi

Sistem informasi adalah kombinasi dari manusia, fasilitas atau sarana teknologi, media, prosedur, dan pengendalian yang ditujukan untuk mengatur jaringan komunikasi, pengolahan transaksi, dan penyajian informasi yang berguna untuk mendukung pengambilan keputusan dalam suatu organisasi. Menurut Laudon dan Laudon (2020), sistem informasi adalah kombinasi terorganisir dari manusia, perangkat keras, perangkat lunak, jaringan komunikasi, dan sumber daya data yang mengumpulkan, mengubah, dan menyebarkan informasi dalam suatu organisasi.

Karakteristik utama sistem informasi meliputi:

1. **Terintegrasi** — Sistem informasi mengintegrasikan berbagai komponen seperti perangkat keras, perangkat lunak, data, prosedur, dan manusia dalam satu kesatuan yang utuh.
2. **Berorientasi pada Pengambilan Keputusan** — Tujuan utama sistem informasi adalah menyediakan informasi yang relevan dan tepat waktu untuk mendukung proses pengambilan keputusan.
3. **Berbasis Komputer** — Sistem informasi modern memanfaatkan teknologi komputer untuk mengolah data secara efisien dan akurat.
4. **Berkelanjutan** — Sistem informasi beroperasi secara berkelanjutan untuk menghasilkan informasi yang dibutuhkan secara terus-menerus.

> **Referensi:** Laudon, K. C., & Laudon, J. P. (2020). *Management Information Systems: Managing the Digital Firm* (16th ed.). New York: Pearson Education.

### 2.1.3 Pengertian Database

Database atau basis data adalah kumpulan data yang terorganisasi dan saling berhubungan yang disimpan secara sistematis sehingga dapat diakses, dikelola, dan diperbarui dengan mudah. Menurut Connolly dan Begg (2015), basis data adalah kumpulan data terkait yang dirancang untuk memenuhi kebutuhan informasi tertentu dari suatu organisasi secara bersama. Sistem Manajemen Basis Data (DBMS) merupakan perangkat lunak yang digunakan untuk mengelola dan mengakses basis data.

Karakteristik dan keunggulan database antara lain:

1. **Terstruktur** — Data disimpan dalam format yang terstruktur dan terorganisir, seperti tabel, baris, dan kolom pada model relasional.
2. **Konsistensi Data** — Database menjamin konsistensi data melalui aturan integritas yang diterapkan pada skema database.
3. **Keamanan** — DBMS menyediakan mekanisme keamanan seperti autentikasi, otorisasi, dan enkripsi untuk melindungi data dari akses tidak sah.
4. **Multi-user Access** — Database modern mendukung akses simultan oleh banyak pengguna tanpa mengorbankan integritas data.
5. **Backup and Recovery** — DBMS menyediakan fasilitas pencadangan dan pemulihan data untuk mengantisipasi kehilangan data akibat kegagalan sistem.

> **Referensi:** Connolly, T., & Begg, C. (2015). *Database Systems: A Practical Approach to Design, Implementation, and Management* (6th ed.). Boston: Pearson Education.

### 2.1.4 Pengertian Internet

Internet adalah jaringan komputer global yang saling terhubung menggunakan standar protokol TCP/IP (*Transmission Control Protocol/Internet Protocol*) yang memungkinkan pertukaran data dan komunikasi antar pengguna di seluruh dunia. Menurut Kurniawan (2018), internet merupakan jaringan komputer raksasa yang menghubungkan jutaan komputer di seluruh dunia tanpa mengenal batas wilayah dan waktu.

Karakteristik dan keunggulan internet meliputi:

1. **Global** — Internet menghubungkan pengguna dari seluruh penjuru dunia tanpa dibatasi oleh batas geografis.
2. **Terbuka** — Internet menggunakan standar protokol terbuka yang memungkinkan siapa saja untuk berpartisipasi dan mengembangkan layanan berbasis internet.
3. **Skalabel** — Infrastruktur internet dapat berkembang dan menampung pertumbuhan jumlah pengguna dan perangkat secara terus-menerus.
4. **Redundansi** — Internet dirancang dengan jalur komunikasi alternatif sehingga jika satu jalur mengalami gangguan, data dapat dialihkan melalui jalur lain.
5. **Layanan Beragam** — Internet mendukung berbagai layanan seperti *World Wide Web* (WWW), surat elektronik (*e-mail*), transfer berkas (*FTP*), dan berbagai protokol komunikasi lainnya.

> **Referensi:** Kurniawan, A. (2018). *Jaringan Komputer*. Yogyakarta: Penerbit Andi.

### 2.1.5 Pengertian UML

UML (*Unified Modeling Language*) adalah bahasa pemodelan standar yang digunakan untuk memvisualisasikan, menspesifikasikan, membangun, dan mendokumentasikan artefak dari suatu sistem perangkat lunak. Menurut Pressman dan Maxim (2020), UML adalah bahasa pemodelan visual yang menyediakan notasi dan sintaks untuk membangun model berorientasi objek dari suatu sistem.

UML memiliki beberapa diagram yang dikelompokkan dalam dua kategori utama, yaitu:

1. **Diagram Struktural** — Meliputi *Class Diagram*, *Object Diagram*, *Component Diagram*, *Deployment Diagram*, *Package Diagram*, *Composite Structure Diagram*, dan *Profile Diagram*. Diagram struktural menggambarkan struktur statis dari suatu sistem.
2. **Diagram Perilaku** — Meliputi *Use Case Diagram*, *Activity Diagram*, *State Machine Diagram*, dan diagram interaksi seperti *Sequence Diagram*, *Communication Diagram*, *Timing Diagram*, dan *Interaction Overview Diagram*. Diagram perilaku menggambarkan dinamika dan perilaku sistem.

Keunggulan UML antara lain:

1. **Standar Industri** — UML merupakan standar yang diakui secara internasional oleh Object Management Group (OMG).
2. **Bahasa Universal** — UML dapat digunakan untuk memodelkan berbagai jenis sistem, tidak terbatas pada domain tertentu.
3. **Komunikasi yang Efektif** — UML menyediakan notasi visual yang memudahkan komunikasi antara pengembang, analis, dan pemangku kepentingan.
4. **Dokumentasi yang Jelas** — Diagram UML berfungsi sebagai dokumentasi teknis yang dapat dipahami oleh berbagai pihak yang terlibat dalam pengembangan sistem.

> **Referensi:** Pressman, R. S., & Maxim, B. R. (2020). *Software Engineering: A Practitioner's Approach* (9th ed.). New York: McGraw-Hill Education.

### 2.1.6 Pengertian Framework

Framework adalah kerangka kerja perangkat lunak yang menyediakan struktur dan komponen dasar untuk memudahkan pengembangan aplikasi. Menurut Fowler (2002), framework merupakan kerangka kerja yang menyediakan kelas-kelas dan fungsi-fungsi yang dapat digunakan kembali (*reusable*) oleh pengembang perangkat lunak untuk membangun aplikasi tertentu. Framework mengimplementasikan pola desain (*design pattern*) dan menyediakan komponen siap pakai yang mempercepat proses pengembangan.

Karakteristik dan keunggulan framework antara lain:

1. **Reusabilitas** — Framework menyediakan komponen dan modul yang dapat digunakan kembali sehingga mengurangi duplikasi kode.
2. **Inversion of Control (IoC)** — Framework mengontrol alur eksekusi aplikasi, dan pengembang hanya perlu mengisi bagian-bagian tertentu yang spesifik.
3. **Standarisasi** — Framework mendorong penerapan standar pengodean dan pola desain yang baik sehingga menghasilkan kode yang lebih terstruktur dan mudah dipelihara.
4. **Efisiensi Pengembangan** — Dengan menyediakan komponen siap pakai, framework mempercepat proses pengembangan aplikasi secara signifikan.
5. **Komunitas dan Ekosistem** — Framework populer memiliki komunitas besar yang menyediakan dokumentasi, plugin, dan dukungan teknis yang melimpah.

> **Referensi:** Fowler, M. (2002). *Patterns of Enterprise Application Architecture*. Boston: Addison-Wesley Professional.

### 2.1.7 Pengertian Bahasa Pemrograman

Bahasa pemrograman adalah sekumpulan instruksi, perintah, dan sintaks yang digunakan untuk memberikan arahan kepada komputer agar menjalankan suatu tugas tertentu. Menurut Sebesta (2016), bahasa pemrograman adalah bahasa formal yang dirancang untuk mengomunikasikan instruksi kepada mesin, khususnya komputer, yang dapat digunakan untuk membuat program yang mengontrol perilaku mesin dan mengekspresikan algoritma secara tepat.

Bahasa pemrograman dapat diklasifikasikan berdasarkan beberapa kategori:

1. **Berdasarkan Tingkat Abstraksi:**
   - *Bahasa Tingkat Rendah* — Bahasa mesin dan bahasa rakitan yang berinteraksi langsung dengan perangkat keras.
   - *Bahasa Tingkat Tinggi* — Bahasa yang lebih mudah dipahami manusia seperti Python, Java, dan PHP.
2. **Berdasarkan Paradigma:**
   - *Pemrograman Prosedural* — Mengorganisasi kode dalam prosedur atau fungsi.
   - *Pemrograman Berorientasi Objek* — Mengorganisasi kode dalam objek-objek yang memiliki properti dan metode.
   - *Pemrograman Fungsional* — Menggunakan fungsi matematatis sebagai konstruksi utama.
3. **Berdasarkan Kompilasi:**
   - *Compiled Language* — Kode diubah menjadi bahasa mesin sebelum dieksekusi.
   - *Interpreted Language* — Kode dieksekusi baris per baris oleh interpreter.

> **Referensi:** Sebesta, R. W. (2016). *Concepts of Programming Languages* (12th ed.). Boston: Pearson Education.

### 2.2.1 Internet of Things (IoT)

Internet of Things (IoT) adalah konsep di mana benda-benda fisik dalam kehidupan sehari-hari dilengkapi dengan sensor, perangkat lunak, dan teknologi jaringan sehingga mampu mengumpulkan dan bertukar data dengan perangkat lain melalui jaringan internet tanpa memerlukan interaksi manusia secara langsung. Menurut Atzori et al. (2010), IoT merupakan paradigma yang berkembang pesat berkat kemajuan teknologi nirkabel, mikro-elektromekanikal, dan internet. Konsep ini didasarkan pada gagasan bahwa setiap benda di dunia dapat diidentifikasi dan diakses melalui jaringan internet.

Karakteristik utama IoT meliputi:

1. **Konektivitas** — Perangkat IoT saling terhubung melalui berbagai protokol komunikasi seperti Wi-Fi, Bluetooth, Zigbee, dan LoRa.
2. **Sensor dan Aktuator** — Perangkat IoT dilengkapi sensor untuk mengumpulkan data dari lingkungan sekitar dan aktuator untuk melakukan tindakan berdasarkan data tersebut.
3. **Skalabilitas** — Arsitektur IoT dirancang untuk mendukung jutaan bahkan miliaran perangkat yang terhubung secara bersamaan.
4. **Otomatisasi** — IoT memungkinkan proses otomatisasi di mana perangkat dapat mengambil keputusan dan bertindak tanpa campur tangan manusia.
5. **Pemrosesan Data Real-time** — IoT memungkinkan pengumpulan, transmisi, dan pemrosesan data secara *real-time* untuk pengambilan keputusan yang cepat.

Dalam konteks proyek ini, IoT digunakan untuk memantau kondisi lingkungan secara *real-time* melalui sensor yang terpasang pada mikrokontroler ESP32 dan mengirimkan data tersebut ke server pusat untuk diproses dan disajikan kepada pengguna melalui antarmuka web.

> **Referensi:** Atzori, L., Iera, A., & Morabito, G. (2010). "The Internet of Things: A survey." *Computer Networks*, 54(15), 2787–2805.

### 2.2.2 MQTT (Message Queuing Telemetry Transport)

MQTT (*Message Queuing Telemetry Transport*) adalah protokol komunikasi ringan berbasis model *publish/subscribe* yang dirancang khusus untuk komunikasi mesin-ke-mesin (*Machine-to-Machine/M2M*) dan lingkungan IoT dengan bandwidth terbatas dan jaringan yang tidak stabil. Protokol ini pertama kali dikembangkan oleh Andy Stanford-Clark dari IBM pada tahun 1999 dan kemudian distandarisasi oleh OASIS (*Organization for the Advancement of Structured Information Standards*) pada tahun 2014 (Banks & Gupta, 2014).

Karakteristik dan keunggulan MQTT antara lain:

1. **Ringan dan Efisien** — MQTT menggunakan *header* yang sangat kecil (minimal 2 byte) sehingga sangat efisien dalam penggunaan bandwidth dan sumber daya perangkat.
2. **Model Publish/Subscribe** — Komunikasi menggunakan model *publish/subscribe* di mana pengirim pesan (*publisher*) dan penerima pesan (*subscriber*) tidak perlu mengetahui satu sama lain secara langsung. Komunikasi dimediasi oleh broker MQTT.
3. **Quality of Service (QoS)** — MQTT menyediakan tiga tingkat QoS:
   - *QoS 0* — Pesan dikirim paling banyak sekali (*at most once*), tanpa jaminan pengiriman.
   - *QoS 1* — Pesan dikirim paling sedikit sekali (*at least once*), dengan jaminan penerimaan tetapi kemungkinan duplikasi.
   - *QoS 2* — Pesan dikirim tepat sekali (*exactly once*), dengan jaminan tanpa duplikasi.
4. **Last Will and Testament** — Fitur ini memungkinkan broker untuk mengirimkan pesan otomatis ketika klien terputus secara tidak terduga.
5. **Retained Messages** — Broker dapat menyimpan pesan terakhir pada suatu *topic* sehingga klien baru yang berlangganan akan langsung menerima pesan terbaru.

Dalam arsitektur proyek Sentinel-IoT, MQTT digunakan sebagai protokol komunikasi utama antara perangkat ESP32 (sebagai *publisher*) dan server backend (sebagai *subscriber*) untuk transmisi data sensor secara *real-time*.

> **Referensi:** Banks, A., & Gupta, R. (Eds.). (2014). *MQTT Version 3.1.1*. OASIS Standard. | Hunkeler, U., Truong, H. L., & Stanford-Clark, A. (2008). "MQTT-S — A publish/subscribe protocol for Wireless Sensor Networks." *Proceedings of the 3rd International Conference on Communication Systems Software and Middleware*, 791–798.

### 2.2.3 ESP32

ESP32 adalah sistem pada chip (*System-on-Chip/SoC*) berbasis mikrokontroler yang dikembangkan oleh Espressif Systems. ESP32 merupakan penerus dari ESP8266 dengan peningkatan signifikan pada aspek konektivitas nirkabel, kinerja pemrosesan, dan fitur-fitur bawaan. Menurut dokumentasi resmi Espressif Systems (2023), ESP32 mengintegrasikan prosesor Xtensa® dual-core 32-bit LX6 yang beroperasi pada frekuensi hingga 240 MHz, serta dilengkapi dengan konektivitas Wi-Fi dan Bluetooth/BLE (*Bluetooth Low Energy*).

Karakteristik dan keunggulan ESP32 antara lain:

1. **Prosesor Dual-Core** — ESP32 menggunakan prosesor Xtensa® LX6 dual-core dengan frekuensi hingga 240 MHz, mampu menjalankan komputasi yang cukup kompleks untuk aplikasi IoT.
2. **Konektivitas Nirkabel Terintegrasi** — ESP32 mendukung Wi-Fi (802.11 b/g/n) dan Bluetooth/BLE 4.2 secara terintegrasi dalam satu chip, sehingga tidak memerlukan modul tambahan.
3. **Antarmuka Periferal yang Lengkap** — ESP32 dilengkapi dengan berbagai antarmuka periferal seperti ADC (*Analog-to-Digital Converter*), DAC (*Digital-to-Analog Converter*), SPI, I2C, UART, dan PWM (*Pulse Width Modulation*).
4. **Mode Operasi Hemat Daya** — ESP32 mendukung berbagai mode hemat daya seperti *Deep Sleep*, *Light Sleep*, dan *Modem Sleep* yang memungkinkan operasi jangka panjang dengan sumber daya baterai.
5. **Kapasitas Memori** — ESP32 dilengkapi dengan SRAM sebesar 520 KB dan mendukung penyimpanan *flash* hingga 16 MB.
6. **Harga Terjangkau** — Dengan fitur yang lengkap, ESP32 ditawarkan dengan harga yang relatif terjangkau sehingga cocok untuk pengembangan proyek IoT berskala besar.

Dalam proyek Sentinel-IoT, ESP32 berfungsi sebagai mikrokontroler utama yang mengumpulkan data dari sensor DHT22 dan mengirimkannya ke server melalui protokol MQTT.

> **Referensi:** Espressif Systems. (2023). *ESP32 Series Datasheet*. Shanghai: Espressif Systems (Shanghai) Co., Ltd. | Kolban, N. (2018). *Kolban's Book on ESP32*. Leanpub.

### 2.2.4 Sensor DHT22

Sensor DHT22 adalah sensor digital yang mampu mengukur suhu dan kelembapan udara secara simultan. Sensor ini diproduksi oleh Aosong Electronics dan dikenal juga dengan nama AM2302. DHT22 menggunakan kapasitor polimer untuk mengukur kelembapan dan termistor (*NTC — Negative Temperature Coefficient*) untuk mengukur suhu, kemudian mengubahnya menjadi sinyal digital melalui konverter ADC internal (Aosong Electronics, 2020).

Karakteristik dan keunggulan sensor DHT22 antara lain:

1. **Rentang Pengukuran yang Luas:**
   - Suhu: -40°C hingga +80°C dengan akurasi ±0.5°C
   - Kelembapan: 0% hingga 100% RH dengan akurasi ±2% RH
2. **Resolusi Tinggi** — DHT22 memiliki resolusi pengukuran sebesar 0.1°C untuk suhu dan 0.1% RH untuk kelembapan.
3. **Output Digital** — DHT22 menghasilkan output digital yang telah terkalibrasi pabrik sehingga tidak memerlukan kalibrasi tambahan oleh pengguna.
4. **Konsumsi Daya Rendah** — Sensor ini memerlukan tegangan operasi 3.3V–6V dengan konsumsi arus yang sangat rendah saat tidak melakukan pengukuran.
5. **Antarmuka Sederhana** — DHT22 menggunakan antarmuka *single-wire* yang hanya memerlukan satu pin data untuk komunikasi dengan mikrokontroler.
6. **Harga Ekonomis** — DHT22 menawarkan keseimbangan antara akurasi dan harga yang sangat kompetitif untuk aplikasi pemantauan lingkungan.

Dalam proyek Sentinel-IoT, sensor DHT22 digunakan untuk memantau kondisi suhu dan kelembapan lingkungan secara *real-time*. Data yang dikumpulkan oleh sensor ini kemudian dikirimkan oleh ESP32 ke server melalui protokol MQTT untuk diproses dan disajikan kepada pengguna.

> **Referensi:** Aosong Electronics. (2020). *AM2302/DHT22 Digital Temperature and Humidity Sensor Datasheet*. Guangzhou: Aosong (Guangzhou) Electronics Co., Ltd.

### 2.2.5 Framework Laravel

Laravel adalah framework aplikasi web berbasis PHP yang bersifat *open-source* dan menggunakan pola arsitektur MVC (*Model-View-Controller*). Laravel pertama kali dikembangkan oleh Taylor Otwell pada tahun 2011 sebagai alternatif dari framework CodeIgniter yang pada saat itu belum mendukung fitur autentikasi dan otorisasi bawaan. Menurut Otwell (2023), Laravel bertujuan untuk membuat proses pengembangan aplikasi web menjadi lebih menyenangkan dan kreatif bagi pengembang tanpa mengorbankan fungsionalitas.

Karakteristik dan keunggulan Laravel antara lain:

1. **Eloquent ORM** — Laravel menyediakan *Object-Relational Mapping* yang intuitif dan elegan untuk berinteraksi dengan basis data menggunakan sintaks berorientasi objek.
2. **Blade Template Engine** — Laravel menggunakan Blade sebagai mesin templating yang memungkinkan pengembang menulis kode HTML dengan sintaks PHP yang bersih dan ekspresif.
3. **Artisan CLI** — Laravel dilengkapi dengan *command-line interface* bernama Artisan yang menyediakan perintah-perintah otomatis untuk mempercepat proses pengembangan seperti migrasi basis data, pembuatan model, dan pengujian.
4. **Sistem Routing yang Ekspresif** — Laravel menyediakan sistem *routing* yang fleksibel dan mudah dibaca untuk mengelola URL aplikasi.
5. **Middleware** — Laravel mendukung mekanisme *middleware* yang memungkinkan pemfilteran permintaan HTTP sebelum sampai ke aplikasi, sangat berguna untuk autentikasi dan otorisasi.
6. **Ekosistem yang Kaya** — Laravel memiliki ekosistem yang luas meliputi Laravel Sail (Docker), Laravel Sanctum (autentikasi API), Laravel Queue (antrian tugas), dan lainnya.
7. **Keamanan Bawaan** — Laravel menyediakan perlindungan terhadap serangan *SQL Injection*, *Cross-Site Request Forgery* (CSRF), dan *Cross-Site Scripting* (XSS) secara bawaan.

Dalam proyek Sentinel-IoT, Laravel digunakan sebagai *backend framework* untuk membangun RESTful API yang mengelola data sensor, autentikasi pengguna, dan komunikasi dengan basis data PostgreSQL.

> **Referensi:** Otwell, T. (2023). *Laravel Documentation*. Laravel. https://laravel.com/docs | Shalaby, M. (2022). *Laravel: Up & Running* (3rd ed.). Sebastopol: O'Reilly Media.

### 2.2.6 React

React adalah pustaka (*library*) JavaScript *open-source* yang dikembangkan oleh Meta (sebelumnya Facebook) untuk membangun antarmuka pengguna (*user interface*) berbasis komponen, khususnya untuk aplikasi web *single-page* (SPA). React pertama kali diperkenalkan oleh Jordan Walke pada tahun 2013 dan saat ini menjadi salah satu pustaka front-end yang paling populer dan banyak digunakan di industri pengembangan perangkat lunak (Abramov & Clark, 2023).

Karakteristik dan keunggulan React antara lain:

1. **Arsitektur Berbasis Komponen** — React memungkinkan pengembang untuk membangun antarmuka pengguna dari komponen-komponen kecil yang dapat digunakan kembali (*reusable*), dikelola secara independen, dan disusun untuk membentuk antarmuka yang kompleks.
2. **Virtual DOM** — React menggunakan Virtual DOM untuk meminimalkan manipulasi DOM secara langsung. Perubahan pada antarmuka pertama-tama direpresentasikan pada Virtual DOM, kemudian React menghitung perbedaan (*diffing*) dan memperbarui hanya bagian yang berubah pada DOM sesungguhnya.
3. **JSX (JavaScript XML)** — React menggunakan JSX, ekensi sintaks JavaScript yang memungkinkan penulisan kode HTML di dalam JavaScript, sehingga meningkatkan keterbacaan dan produktivitas pengembang.
4. **Unidirectional Data Flow** — React mengikuti alur data satu arah (*one-way data binding*) yang memudahkan *debugging* dan pemahaman alur data dalam aplikasi.
5. **Ekosistem yang Luas** — React memiliki ekosistem yang sangat kaya meliputi Next.js (framework SSR), Redux (manajemen state), React Router (navigasi), dan berbagai pustaka UI lainnya.
6. **React Hooks** — Fitur *hooks* seperti `useState`, `useEffect`, dan `useContext` memungkinkan pengelolaan *state* dan efek samping (*side effects*) pada komponen fungsional.

Dalam proyek Sentinel-IoT, React digunakan untuk membangun antarmuka pengguna (*frontend*) yang interaktif dan responsif untuk menampilkan data sensor, grafik pemantauan, dan panel kontrol sistem.

> **Referensi:** Abramov, D., & Clark, A. (2023). *React Documentation*. Meta Open Source. https://react.dev | Banks, A., & Porcello, E. (2020). *Learning React* (2nd ed.). Sebastopol: O'Reilly Media.

### 2.2.7 PostgreSQL

PostgreSQL adalah sistem manajemen basis data relasional (*Relational Database Management System/RDBMS*) *open-source* yang terkenal akan keandalan, fitur yang lengkap, dan kepatuhan terhadap standar SQL. PostgreSQL pertama kali dikembangkan oleh Michael Stonebraker di University of California, Berkeley pada tahun 1986 dengan nama POSTGRES, dan kemudian berganti nama menjadi PostgreSQL pada tahun 1996 (PostgreSQL Global Development Group, 2023).

Karakteristik dan keunggulan PostgreSQL antara lain:

1. **Kepatuhan Standar SQL** — PostgreSQL mendukung sebagian besar fitur standar SQL:2016, menjadikannya salah satu RDBMS yang paling patuh terhadap standar industri.
2. **ACID Compliance** — PostgreSQL menjamin properti ACID (*Atomicity, Consistency, Isolation, Durability*) pada setiap transaksi sehingga memastikan integritas data.
3. **Tipe Data yang Kaya** — PostgreSQL mendukung berbagai tipe data bawaan seperti JSON, JSONB, Array, HStore, UUID, dan tipe data spasial melalui ekstensi PostGIS.
4. **Kemampuan Ekstensi** — PostgreSQL menyediakan arsitektur ekstensi yang memungkinkan penambahan fungsionalitas baru tanpa memodifikasi inti sistem.
5. **Concurrency Control** — PostgreSQL menggunakan teknik *Multi-Version Concurrency Control* (MVCC) yang memungkinkan akses simultan oleh banyak pengguna tanpa saling memblokir.
6. **Keamanan** — PostgreSQL menyediakan mekanisme keamanan yang komprehensif meliputi autentikasi berbasis sertifikat, autentikasi LDAP, kontrol akses berbasis peran (*Role-Based Access Control*), dan enkripsi data.
7. **Skalabilitas** — PostgreSQL mendukung replikasi sinkron dan asinkron, partisi tabel, serta indeks lanjutan untuk menangani volume data yang besar.

Dalam proyek Sentinel-IoT, PostgreSQL digunakan sebagai basis data utama untuk menyimpan data sensor, data pengguna, konfigurasi perangkat, dan log aktivitas sistem.

> **Referensi:** PostgreSQL Global Development Group. (2023). *PostgreSQL 16 Documentation*. https://www.postgresql.org/docs/16/ | Douglas, K., & Douglas, S. (2019). *PostgreSQL* (3rd ed.). Sebastopol: O'Reilly Media.

### 2.2.8 Docker

Docker adalah platform *open-source* yang menyediakan layanan virtualisasi pada tingkat sistem operasi (*OS-level virtualization*) untuk mengemas aplikasi dan dependensinya ke dalam wadah (*container*) yang ringan dan portabel. Docker pertama kali dikembangkan oleh Solomon Hykes dan dirilis pada tahun 2013 oleh perusahaan Docker, Inc. Menurut Merkel (2014), Docker menyediakan abstraksi tambahan dan otomatisasi virtualisasi pada tingkat sistem operasi untuk menyediakan *container* yang portabel dan mandiri.

Karakteristik dan keunggulan Docker antara lain:

1. **Isolasi** — Setiap *container* berjalan secara terisolasi dengan *filesystem*, jaringan, dan prosesnya sendiri, sehingga aplikasi yang berbeda tidak saling mengganggu.
2. **Portabilitas** — *Container* Docker dapat berjalan di mana saja — laptop lokal, server fisik, maupun layanan *cloud* — tanpa memerlukan modifikasi.
3. **Ringan** — *Container* berbagi *kernel* sistem operasi yang sama sehingga lebih ringan dibandingkan dengan mesin virtual (*virtual machine*) yang memerlukan sistem operasi tersendiri.
4. **Dockerfile** — Docker menggunakan Dockerfile sebagai *blueprint* yang mendefinisikan langkah-langkah pembangunan (*build*) *image* secara terstruktur dan dapat direproduksi.
5. **Docker Compose** — Docker menyediakan Docker Compose untuk mendefinisikan dan menjalankan aplikasi multi-*container* dengan satu berkas konfigurasi YAML.
6. **Ekosistem yang Luas** — Docker Hub menyediakan repositori publik yang berisi ribuan *image* siap pakai yang dapat diunduh dan digunakan secara langsung.
7. **Orkestrasi** — Docker mendukung orkestrasi *container* melalui Docker Swarm maupun integrasi dengan Kubernetes untuk pengelolaan *container* dalam skala besar.

Dalam proyek Sentinel-IoT, Docker digunakan untuk mengontainerisasi seluruh komponen sistem meliputi aplikasi Laravel (backend), aplikasi React (frontend), broker MQTT (Mosquitto), dan basis data PostgreSQL, sehingga memudahkan proses pengembangan, pengujian, dan penyebaran (*deployment*) aplikasi.

> **Referensi:** Merkel, D. (2014). "Docker: Lightweight Linux Containers for Consistent Development and Deployment." *Linux Journal*, 2014(239), 2. | Mouat, A. (2015). *Using Docker*. Sebastopol: O'Reilly Media.

### 2.2.9 Tailwind CSS

Tailwind CSS adalah kerangka kerja (*framework*) CSS utilitas-first (*utility-first*) yang menyediakan kelas-kelas CSS siap pakai untuk membangun desain antarmuka pengguna secara langsung di dalam markup HTML. Tailwind CSS dikembangkan oleh Adam Wathan dan pertama kali dirilis pada tahun 2017. Berbeda dengan framework CSS tradisional seperti Bootstrap yang menyediakan komponen siap pakai, Tailwind CSS menyediakan kelas-kelas utilitas atomik (*atomic utility classes*) yang dapat dikombinasikan untuk membentuk desain kustom tanpa perlu menulis CSS dari awal (Wathan, 2023).

Karakteristik dan keunggulan Tailwind CSS antara lain:

1. **Utility-First** — Tailwind CSS menyediakan kelas utilitas atomik seperti `flex`, `pt-4`, `text-center`, dan `bg-blue-500` yang dapat dikombinasikan langsung dalam elemen HTML untuk membentuk desain yang diinginkan.
2. **Responsif** — Tailwind CSS menggunakan pendekatan *mobile-first* dengan *breakpoint* yang dapat dikustomisasi (`sm:`, `md:`, `lg:`, `xl:`, `2xl:`) untuk membangun desain responsif secara intuitif.
3. **Kustomisasi Tinggi** — Konfigurasi Tailwind CSS (`tailwind.config.js`) memungkinkan pengembang untuk menyesuaikan palet warna, tipografi, jarak, *breakpoint*, dan berbagai aspek desain lainnya sesuai kebutuhan proyek.
4. **Tree-Shaking** — Tailwind CSS secara otomatis menghapus kelas-kelas utilitas yang tidak digunakan (*purging*) pada saat proses produksi, sehingga menghasilkan berkas CSS yang sangat kecil dan optimal.
5. **Dark Mode** — Tailwind CSS mendukung mode gelap (*dark mode*) secara bawaan melalui kelas `dark:`.
6. **Responsive Design** — Mendukung desain responsif dengan *utility-first approach* yang memungkinkan penyesuaian tampilan pada berbagai ukuran layar.
7. **Konsistensi Desain** — Dengan menggunakan kelas utilitas yang sudah didefinisikan, Tailwind CSS mendorong konsistensi desain di seluruh aplikasi.

Dalam proyek Sentinel-IoT, Tailwind CSS digunakan sebagai framework CSS utama untuk membangun antarmuka pengguna yang responsif, modern, dan konsisten pada aplikasi frontend yang dibangun dengan React.

> **Referensi:** Wathan, A. (2023). *Tailwind CSS Documentation*. Tailwind Labs. https://tailwindcss.com/docs | Gagliardi, D. (2022). "Tailwind CSS: From Zero to Production." *Frontend Masters*.
