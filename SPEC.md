**C. AI Agent realtime chat** ŌĆö `resources/js/pages/agent/index.tsx`
- Critical requirement: feels like ChatGPT/Claude (token streaming, typing

  (`AgentMessageCompleted`) so other parts of the app (and external webhooks
  configured via env `AGENT_WEBHOOK_URL`) can subscribe later. Even if the
  external webhook is unused now, plumb the event class.

## 4. Constraints (hard)

1. **Loop on error** ŌĆö if `npm run build` or Pest tests fail, debug and re-run
   until green. Don't stop with a broken build.
2. **No file deletion outside scope** ŌĆö only write or modify files inside
   `app/`, `resources/`, `routes/`, `tests/`, `docs/`, `README.md`,
   `AGENTS.md`. Do not touch `bin/rotate-bot-token.sh`, `_diagnose_token.php`,
   `.env`, `composer.json`/`package.json` lock files unless adding a single
   tested dep.
3. **Reuse before create** ŌĆö check existing components in
   `resources/js/components/` and `resources/js/components/ui/` before
   creating new ones. shadcn already provides table/badge/button/etc.
4. **Wayfinder typed routes** ŌĆö keep using `@/routes/<resource>` helpers, never
   hand-write URLs.
5. **Inertia v3 patterns** ŌĆö `Inertia::render`, `useForm`, `useHttp` for
   ad-hoc, optimistic updates, deferred props. No manual axios.
6. **No emoji as icons** ŌĆö use lucide-react.
7. **Accessibility** ŌĆö keyboard nav, focus rings, aria-labels for icon-only
   buttons, 4.5:1 contrast minimum, prefers-reduced-motion respected.
8. **Performance** ŌĆö Lighthouse mobile target Ōēź 80 on Welcome and Dashboard.
9. **No interactive prompts** ŌĆö every step must be runnable headlessly. Use
   `--force` / `--no-interaction` on artisan; pin all installs.
10. **Feature flag** ŌĆö if Reverb is needed, gate it behind
    `BROADCAST_DRIVER=reverb` so the existing Docker stack still boots when
    the flag is off.
