## 1. What is Sentinel-IoT

A self-hosted IoT Security Operations Center. Backend = Laravel 13 + Postgres + Mosquitto MQTT + a
SentinelAgent (laravel/ai package) wired to a local LLM. Frontend = Inertia v3 + React 19 +
Tailwind v4 + shadcn/ui. Telegram bot for ChatOps. All running in Docker Compose.

The product lets a security team monitor IoT devices, audit MQTT traffic, triage incidents, and
talk to an AI co-pilot about all of it — in one console.

The current frontend is the Laravel + React starter UI with a partial custom theme. The brief is
to elevate it to a polished cyberpunk-IoT-SOC console without breaking any backend.

### G4. AI Agent realtime chat (CRITICAL FEATURE) — `resources/js/pages/agent/index.tsx`

Current chat is form-post-and-redirect. Replace with token-streaming UX that feels like
ChatGPT/Claude.

**Backend**:
- Add `POST /agent/stream` route returning a streamed response (text/event-stream or chunked
  JSON-lines). Reuse `SentinelAgent::stream()` (already exists in
  `app/Ai/Agents/SentinelAgent.php`). Use Symfony `StreamedResponse`.
- Dispatch a Laravel event `AgentMessageCompleted` when the run finishes. Plumb only — provide
  a stub listener in `app/Listeners/` documenting where to add an external webhook.

  approach.

**Frontend**:
- Use `fetch` + `ReadableStream` (no axios — Inertia v3 removed it). Append tokens as they
  arrive. Show animated typing dots while waiting for first token. Optimistic user message.
  Auto-scroll-to-bottom unless user has scrolled up. Copy + regenerate buttons per assistant
  message.
- Persist final assistant message via the existing `AgentMessage` model (controller writes the
  audit row already).
- Reuse existing components: `Card`, `Button`, `Textarea`, `ScrollArea`, `MarkdownView`.
- Mobile-friendly chat layout (bottom-anchored input, content scrolls above).

## 5. Hard constraints

1. **No file deletion outside scope** — only write/modify under `app/`, `resources/`, `routes/`,
   `tests/`, `docs/`, `README.md`, `AGENTS.md`. Do NOT touch `bin/rotate-bot-token.sh`,
   `.env`, `composer.json/lock`, `package.json/lock` unless you can justify a single new dep.
2. **Reuse before create** — check existing components first. shadcn already provides
   table/badge/button/etc.
3. **Wayfinder typed routes** — keep using `@/routes/<resource>`, never hand-write URLs.

   updates, deferred props. No manual axios.
5. **No emoji as icons** — only lucide-react.
6. **Accessibility** — keyboard nav, focus rings, aria-labels for icon-only buttons, 4.5:1
   contrast minimum, `prefers-reduced-motion` respected.
7. **Performance** — Lighthouse mobile target ≥ 80 on Welcome and Dashboard.
8. **Responsive** — verify visually at 375 / 768 / 1024 / 1440. Touch targets ≥ 44px on mobile.
9. **No interactive prompts** — every step must be runnable headlessly. Use `--force` /
   `--no-interaction` on artisan; pin all installs.
10. **Loop on errors** — if a verify command fails, fix and re-run until green. Don't stop at
    a broken build.

## 10. Reference: design system snapshot

From `design-system/sentinel-iot/MASTER.md`:

- Style: Dark OLED. Background `--sentinel-bg-primary` (`#050b16`).
- Heading font: mono (Geist Mono / Fira Code feel via existing Geist Variable).
- Body font: Geist Variable (already imported via `@fontsource-variable/geist`).
- Accents: teal `#1fe6d0`, cyan `#38bdf8`, purple `#a855f7`.
- Effects: `text-shadow: 0 0 10px <accent>` for headlines, `box-shadow: 0 0 24px <accent>22` for
  glow on hover, `.sentinel-grid-bg` for ambient grid.
- Anti-patterns: emojis as icons, missing `cursor-pointer`, layout-shifting hovers, low
  contrast, instant state changes (always 150–300ms), invisible focus.
